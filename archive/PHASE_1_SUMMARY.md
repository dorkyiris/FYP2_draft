## Deliverables

### 1. Core Biomechanics Module (`rehabilitationcore/`)

#### `models.py` - Immutable Data Models
- **ExerciseStatus** enum: PASS, FAIL, TRANSITIONING, TRACKING
- **Landmark** dataclass: Represents 3D pose landmarks with visibility
- **AngleThreshold**: Threshold configuration with automatic evaluation
- **ExerciseDefinition**: Clinical exercise specification (static, configuration-like)
- **ExerciseResult**: Immutable analysis result for each exercise

**Benefits:**
- Type-safe: No more ambiguous dictionary keys
- Serializable: Can be converted to JSON/protobuf for APIs
- Frozen: Prevents accidental mutations in analysis pipeline

#### `biomechanics.py` - Pure Math Functions
- **calculate_2d_angle()**: Core angle calculation (3 points → degrees)
- **calculate_3d_angle()**: 3D angle support for future enhancement
- **calculate_distance()**: Euclidean distance for landmark positions
- **smooth_signal()**: EMA and SMA smoothing with NaN handling
- **validate_landmark_chain()**: Pre-flight checks before analysis

**Key Properties:**
- Pure functions: No state, no side effects
- Testable: 20 unit tests covering edge cases
- Clinical-grade: Validated against known angles (90°, 180°, etc.)

#### `exercises.py` - Exercise Definitions
- **EXERCISES** registry: All 3 exercises fully defined
- **EXERCISE_1_DEFINITION**: Arm Abduction (elbow ≥ 160°)
- **EXERCISE_2_DEFINITION**: V-to-W Transition (120° to 90° shoulder)
- **EXERCISE_3_DEFINITION**: Inclined Push-up (elbow ≤ 100°)

**Key Features:**
- Landmark indices documented in comments
- Thresholds externalized from code logic
- Feedback rules customizable per exercise
- Clinical parameters version-controllable

#### `analyzer.py` - Exercise Analysis Engine
- **BiomechanicalAnalyzer**: Low-level analysis orchestration
  - Validates landmark chain
  - Calculates all angles
  - Evaluates thresholds
  - Generates feedback
  
- **ExerciseAnalyzer**: High-level API
  - Single-frame analysis: `analyze(landmarks, exercise)`
  - Sequence analysis: `analyze_sequence(landmark_frames, exercise)`
  - Consistent result objects with confidence scores

**Why Separate from UI:**
```python
# Before (old monolithic app.py):
# - Cannot test without Streamlit
# - Cannot reuse in API/mobile
# - UI and logic tightly coupled

# After:
from rehabilitationcore import ExerciseAnalyzer, get_exercise
analyzer = ExerciseAnalyzer()
result = analyzer.analyze(landmarks, get_exercise(1))
# Works in any context: CLI, API, mobile, tests
```

    landmarks = [Landmark(**l) for l in landmarks_json]

## Migration Guide: Old Code → New Code

### Example: Angle Calculation

**Old (app.py):**
```python
def calculate_2d_angle(a, b, c):
    radians = np.arctan2(c[1] - b[1], c[0] - b[0]) - np.arctan2(a[1] - b[1], a[0] - b[0])
    angle = np.abs(radians * 180.0 / np.pi)
    if angle > 180.0: angle = 360.0 - angle
    return angle
```

**New (rehabilitationcore/biomechanics.py):**
```python
from rehabilitationcore import calculate_2d_angle
# Same function, better tested, documented, reusable
angle = calculate_2d_angle(p1, p2, p3)
```
